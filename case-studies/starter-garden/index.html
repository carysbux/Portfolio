Starter Garden

Starter Garden —

A server-rendered beginner gardening site with JSON-backed pages, slideshow and gallery partials, and plant cards filtered per topic (indoors, outdoors, vegetables) — so content can grow without rewriting route logic.

Express Handlebars JSON content
Website screenshot

Problem

New gardeners need short, trustworthy recommendations — not endless generic articles. The site also had to stay easy to extend: more plants, new sections, and refreshed imagery should not require duplicating HTML across every page.

Goal

Ship a small full-stack style experience with Express and Handlebars where copy, navigation, slideshow slides, gallery items, and plant records live in JSON. A single dynamic route should resolve each topic page and attach the right slides and plant cards by a shared page key.

Solution

The app loads navigation, slideshow, gallery, pages, and plants at startup. GET / renders the home carousel from slides flagged home: true; GET /page/:page looks up metadata in pages.json, filters slides and plants by that slug, and passes everything into one layout template with reusable partials.

Features & Benefits

Feature Function Benefit
JSON-driven pages Titles and descriptions live in pages.json keyed by route slug. Update copy or add sections with minimal template edits.
Filtered slideshow Each slide declares home and/or page so the carousel matches the current topic. Home and topic pages stay visually on-message without duplicate templates.
Plant grid partial Plant objects include page, name, description, and image path; Handlebars loops render Bootstrap cards. Consistent card UI while the dataset grows.
Static assets express.static serves CSS, JS, and images under public/ (with a /public alias). Simple hosting story and predictable image URLs in JSON.

Technical Challenges

  • Keeping slideshow page values aligned with route slugs (indoors, outdoors, veggies, about) so filters return the intended slides.
  • Normalizing plant page keys when matching the URL parameter (trim and case) to avoid empty grids from data drift.
  • Image paths in JSON must stay consistent with how static files are exposed (public/... prefixes).

Next Steps

  • Deploy to a small host (e.g. Render, Railway) with a production PORT and clear run instructions for whoever operates it.
  • Add search or tag filters on plant-heavy pages so beginners can narrow choices quickly.
  • Optional admin or build step to validate JSON (missing images, orphan slugs) before deploy.
